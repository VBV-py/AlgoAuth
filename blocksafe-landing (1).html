CLIENT-SIDE ENCRYPTION

Files are encrypted with AES-256-GCM inside your browser. The raw file never touches any server ‚Äî encrypted or not. The encryption key is generated locally and never transmitted in plaintext.

AES-256-GCM
üß©
SHAMIR'S SECRET SHARING

The encryption key is mathematically split into 3 shares distributed across independent Trustless Trio nodes. Any 2 of 3 shares reconstruct the key. No single node can decrypt your file alone.

2-of-3 Threshold
‚õì
ON-CHAIN ACCESS CONTROL

Smart contracts replace access control lists. Group membership, file policies, and timestamps are stored immutably on Ethereum. No admin can quietly edit who accessed what.

Sepolia Testnet
‚è±
TIME-BASED ABAC

Attribute-Based Access Control with temporal enforcement. Access grants expire automatically after a configurable window. Historical access is checked as-of the file's creation timestamp, not today's state.

Auto-Expiring Grants
üåê
IPFS DECENTRALIZED STORAGE

Encrypted files are stored on IPFS ‚Äî the peer-to-peer content-addressed network. No central server to breach. Files are referenced by content hash (CID), making tampering cryptographically impossible.

Content-Addressed

IMMUTABLE AUDIT TRAILS
